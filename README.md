# 《FreeCAD Sketcher Solver Architecture》 - 中文翻译文档

## 文档信息

- **原文档**: FreeCAD Sketcher Solver Architecture.pdf
- **作者**: Abdullah Tahiri
- **发布日期**: December 2018
- **翻译日期**: 2025年3月
- **文档大小**: 65页，5个章节

## 概述

本文档是FreeCAD中求解器架构的技术详解，主要面向以下读者：

1. 在Sketcher中引入新几何图形支持的开发者
2. 在Sketcher中引入新约束支持的开发者
3. 尝试解决求解器bug的开发者
4. 需要从求解器过程向用户提供更多信息的开发者
5. 向求解器引入新算法的开发者
6. 改进求解器算法的开发者

## 章节内容概述

### 第1章：引言
**核心内容**：文档范围、目标读者、FreeCAD Sketcher求解器架构的基本概念

**重点**：
- 本文档的适用范围（不是使用教程，而是架构详解）
- 目标读者：开发者、算法研究者、希望扩展Sketcher的人员
- 五个主要应用场景：新几何/约束支持、bug修复、算法改进等

### 第2章：Sketcher架构
**核心内容**：三层架构设计（Sketcher、求解器接口、求解器）

**重点**：
- SketchObject：Sketcher层，管理几何图形和约束
- Sketch类：求解器接口，负责数据转换
- PlaneGCS：求解器层，执行实际求解
- 几何图形和约束的数据结构设计
- 参数系统：自由度（DoF）计算
- 求解器几何图形：DeriVector2向量和Curve抽象类

### 第3章：Sketcher函数
**核心内容**：Sketcher提供给用户的功能实现

**重点**：
- 求解（solve）与重新计算（recompute）的区别
- Sketch修改的两种模式：求解 vs 重新计算
- 几何图形移动：程序化移动和用户交互拖动
- 求解器信息获取：自由度、冗余约束、冲突约束、依赖参数

### 第4章：求解器接口
**核心内容**：连接Sketcher和PlaneGCS的接口层实现

**重点**：
- 数据结构：GeoDef（几何定义）、ConstrDef（约束定义）、参数向量
- 数据传递流程：从Sketcher到求解器，再从求解器返回Sketcher
- 几何图形转换：内存分配和索引管理
- 约束创建：标签机制和驱动/参考约束处理
- 程序化移动：initMove、movePoint、resetInitMove
- 求解器信息缓冲区：诊断结果的缓存和管理

### 第5章：PlaneGCS求解器
**核心内容**：求解器的核心算法和数据结构

**重点**：
- 约束基类：参数管理、标签系统、误差和梯度计算
- 具体约束实现：以ConstraintDifference为例
- 初始化和诊断：QR分解（稠密和稀疏算法）
- 系统分析：秩计算、依赖参数检测、冗余/冲突约束识别
- 子系统划分：解耦组件识别、参数缩减、等式约束处理
- 求解算法：BFGS、Levenberg-Marquardt、DogLeg、优先级求解
- 求解策略：单个子系统、两个子系统（优先级）、通用求解
